## 3. ブランドアイデンティティ

### トーン

- **信頼** — データに基づく提案。煽らない
- **プロフェッショナル** — コンサルタントの知見がプロダクトに内包されている
- **シンプル** — URL入力だけ。追加入力を要求しない

### カラー

- メインカラー：信頼のブルー（#1B3A5C系）
- アクセント：未定（Phase 0.5のUI構築時に確定）

### ビジュアル方針

- Ahrefs Site Explorer型の情報密度
- チャットUIではなく、ダッシュボード型
- 日本語ファーストのタイポグラフィ

## 4. 「Publish Gate」の名前問題

### 現状分析

- 「Gate」はLP公開判断（Publish/Not Publish）を想起させる
- Phase 0.5-1（LP改善ツール）には適合する
- Phase 2以降（Webマーケダッシュボード、マーケティングOS）ではスコープに合わない可能性

### 判断

- **Phase 1までは「Publish Gate」を維持する**
- 名前変更のリスク（ブランド認知のリセット、SEO資産の喪失、既存ユーザーの混乱）が高い
- Phase 2移行時に再検討（論点G）

---

## 5. ビジネスモデルとブランドの関係

### Ahrefsモデル（ブートストラップ、利益重視）

- VCからの調達なし。利益でプロダクトを成長させる
- ブランドメッセージ：「プロダクトの品質で選ばれる」
- マーケティング予算は限定的。プロダクト主導の成長（PLG）
- コンテンツマーケティング（SEO記事）が主要な認知チャネル

### ブランドへの影響

- 「急成長スタートアップ」の印象を与えない
- 「堅実に価値を提供するツール」の印象を構築
- 価格設定は利益を確保できる水準（薄利多売しない）

## 6. ポジショニング戦略

### Phase別の競合カテゴリ変更

各Phase移行時にカテゴリを変えることで、直接比較を回避する。

| Phase | 「〇〇と比べてどう？」への回答 |
|-------|---------------------------|
| 0.5-1 | 「dejamより安く（Free入口あり）、しかもLP改善だけでなく広告訴求まで出る」 |
| 2 | 「AhrefsのようにURLから始まるが、SEOだけでなくマーケティング全体が見える」 |
| 3 | 「HubSpotのようなオールインワンだが、導入コストがゼロ（URL入力だけ）」 |

### 避けるべきポジショニング

- 「AIツール」という汎用カテゴリ → 「薄いAIラッパー」と認識されるリスク
- 「LP改善ツール」に固定 → Phase 2以降のスコープ拡張が困難に
- 「開発者名」を前面に → 独立プロダクトとしてのブランド毀損（論点Hで要検討）
