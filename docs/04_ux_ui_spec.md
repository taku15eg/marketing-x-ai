# Publish Gate UX/UIデザイン仕様書

**版：v3.0　更新日：2026-03-08（全面刷新）**

---

## 1. UX方針：Ahrefs Site Explorer型

**URLを入れるだけで全部出る。チャットUIではない。**

| 時間 | 体験 | 場所 |
|------|------|------|
| 0秒 | Webダッシュボードにアクセス | Web |
| 3秒 | URL入力バーにURLをペースト | Web |
| 5秒 | 分析開始（4ステップパイプライン） | Web |
| 15-30秒 | **結果一覧が表示。タブ1（LP分析）の課題一覧** | Web |
| 60秒 | 課題の詳細を展開。「なるほど」 | Web |
| 90秒 | 依頼書を生成→コピー or 共有URLを生成 | Web |
| 翌日 | Chrome拡張のアイコンが光る→「別のLPも」 | Chrome拡張 |
| 1週間後 | タブ2（広告訴求）を試す→Starter検討 | Web |

**この体験にアカウント不要（Free月5回）。追加入力不要。チュートリアル不要。**

## 2. デザイン原則

| # | 原則 | v3.0でのルール |
|---|------|---------------|
| 1 | URL入力が主人公 | ダッシュボード最上部に常時URL入力バー |
| 2 | 重要3点 | 初期表示は課題3つまで。深掘りはアコーディオン |
| 3 | 段階的開放 | タブ1のみアクティブ。他タブはロックアイコン＋「Starterで解放」 |
| 4 | 浦田型を圧倒しない | 6タブを同時に見せない。段階的に解放 |
| 5 | 1つの主CTA | 各画面で押すべきボタンは原則1つ |
| 6 | 次にやることが必ず付く | 「分析ツール止まり」回避。課題→改善方向性→依頼書 |
| 7 | 信頼の可視化 | 根拠ラベル（ページ情報/Vision/推定/外部データ）を表示 |
| 8 | 共有を促す | 「この結果を共有」ボタンを結果ページの目立つ位置に |
