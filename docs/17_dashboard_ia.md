# Publish Gate Webダッシュボード IA設計書

**版：v3.0　作成日：2026-03-08**

---

## 1. 情報アーキテクチャ

### ページ構造

```
/                          トップ（URL入力、最近の分析一覧）
/analysis/:id              分析結果（6タブ構造）
/analysis/:id/handoff      依頼書（デザイナー/エンジニア切替）
/share/:id                 共有URL閲覧ページ（Read-Only、アカウント不要、CTA付き）
/pricing                   プラン選択（Free/Starter/Pro/Business）
/settings                  設定（アカウント、プラン管理、GSC/GA4連携）
/settings/integrations     Google APIs連携
```

### 分析結果ページ（6タブ構造）

| タブ | 内容 | プラン | データソース | Phase |
|------|------|--------|------------|-------|
| タブ1: LP分析 | 課題一覧→展開→依頼書→薬機法 | Free〜 | URL（Vision+DOM） | 0.5 |
| タブ2: 広告訴求 | Google/Meta/PMax訴求文 | Starter〜 | LP分析結果から逆算 | 1 |
| タブ3: 市場分析 | GSC→クエリ→ボリューム→傾向 | Pro〜 | GSC+キーワードプランナー | 2 |
| タブ4: 流入分析 | GA4→参照元→セッション→媒体別 | Pro〜 | GA4 | 2 |
| タブ5: 競合分析 | 2URL比較→構造差分 | Pro〜 | AI推定+公開情報 | 2 |
| タブ6: 事業分析 | 市場規模→顧客単価→事業モデル | Business〜 | AI推定+業界データ | 3 |

## 2. 各ページ詳細

### `/` トップページ

- URL入力フォーム（中央配置、最も目立つ要素）
- 最近の分析一覧（ログイン時のみ表示）
- サンプル分析結果へのリンク（未ログイン時）
- 「URLを入れるだけ」のメッセージ

### `/analysis/:id` 分析結果ページ

- 6タブのタブバー（上部固定）
- タブ1がデフォルト表示
- 未開放タブはロック表示（プラン名とアップグレードCTA）
- 共有ボタン（右上、目立つ位置）
- 依頼書生成ボタン（タブ1内、課題展開後に表示）
- 再分析ボタン

### `/analysis/:id/handoff` 依頼書ページ

- デザイナー向け/エンジニア向けの切替タブ
- 構造化された依頼内容（課題→対応方針→具体指示）
- 計測設計（実装後のKPI設定）
- 薬機法チェック結果（該当する場合）
- クリップボードコピーボタン
- フッターに「Powered by Publish Gate」

### `/share/:id` 共有URL閲覧ページ

- Read-Only表示（編集不可）
- アカウント不要で閲覧可能
- 表示内容：分析結果のスナップショット
- CTA：「自分のURLでも分析する」→ `/` へ遷移
- CTA：「無料で始める」→ アカウント作成
- フッターに「Publish Gateで作成」

### `/pricing` プラン選択ページ

- 4プラン横並び（Free/Starter/Pro/Business）
- 各プランの機能比較表
- 現在のプラン表示（ログイン時）
- Starter/Pro/Businessの申し込みボタン

### `/settings` 設定ページ

- アカウント情報
- プラン管理（現在のプラン、アップグレード/ダウングレード）
- `/settings/integrations` への導線

### `/settings/integrations` Google APIs連携ページ

- GSC連携（OAuth、「読み取りのみ」明記）
- GA4連携（OAuth、「読み取りのみ」明記）
- 連携状態の表示（接続済み/未接続）
- 接続解除ボタン

## 3. 導線設計

### メイン導線

```
URL入力（/）→ 分析結果（/analysis/:id）→ タブ1がデフォルト → 共有URL生成 or 依頼書生成
```

### タブ間ストーリー接続

タブ間は独立ではなく、ストーリーでつながる。

- タブ1（LP分析）の課題から → 「この課題を広告でも解決するなら？」→ タブ2へ誘導
- タブ2（広告訴求）の結果から → 「この訴求が効く市場は？」→ タブ3へ誘導
- タブ3（市場分析）の結果から → 「どこから流入している？」→ タブ4へ誘導
- タブ4（流入分析）の結果から → 「競合はどうしている？」→ タブ5へ誘導

### 段階的開放

- Phase 0.5：タブ1のみ有効。タブ2-6はロック表示（プラン名表示）
- Phase 1：タブ1-2が有効（Starter以上）
- Phase 2：タブ1-5が有効（Pro以上）
- Phase 3：タブ1-6が有効（Business）

ロックされたタブは「何が見えるか」のプレビューを表示し、アップグレードの動機を作る。

### 共有ボタンの配置

- 分析結果ページの右上（常時表示、目立つ色）
- 各タブの結果セクション末尾にも小さい共有ボタン
- 依頼書ページにも共有ボタン

## 4. レスポンシブ設計

| デバイス | 対応方針 |
|---------|---------|
| デスクトップ | メイン。Ahrefs型の情報密度 |
| タブレット | タブを横スクロール対応 |
| モバイル | 共有URL閲覧ページ（/share/:id）のみ最適化。分析はデスクトップ推奨 |

共有URLの閲覧はモバイルで行われる可能性が高い（Slackで受け取って即開く）ため、`/share/:id` のモバイル対応は必須。
