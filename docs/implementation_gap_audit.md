## 買いたい状態

- **需要側**: マーケ担当が、LPのURLを1つ入れるだけで、「何が悪いのか」「誰に何を依頼すればよいか」が不安なく分かり、その場でデザイナー/エンジニアに共有できる
- **発注側**: 改善の方向性・優先順位・依頼書が揃っていれば、専門知識なしで判断・発注できる
- **実行側**: 構造化された依頼パックと計測設計が揃えば、再現性高くLP改善を回せる

## 避けたい状態

- 分析結果が共有されず1人で閉じる → バイラルが起きない
- 分析は出るが抽象的で意思決定に繋がらない → 行動可能性の欠如
- 一度使って終わる → リテンション不在
- セキュリティや法令で止まる → 信頼毀損
- Chrome拡張が主語になる → Web本体の価値が薄まる

## 意思決定の詰まり（現フェーズ）

1. **永続化の不在**: In-memoryストアのみでサーバー再起動で全データ消失。β検証には耐えるが、共有URLの信頼性が低い
2. **Vision APIの任意性**: SCREENSHOT_API_KEY未設定時はDOM-onlyにフォールバック。品質劣化の許容範囲が未定義
3. **企業リサーチの浅さ**: Step 1がHTML解析のみで、業界分類・規制フラグ・事業モデル推定がClaude依存（Step 3-4に混在）
