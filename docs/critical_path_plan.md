# クリティカルパス計画 — Phase 0.5 Sprint

**作成日**: 2026-03-14

---

## 本質論点

Publish Gateの Phase 0.5 MVP は既に 85-90% 実装済み。
残るクリティカルパスは「動くこと」ではなく「**安全に動き、計測でき、共有価値が高品質であること**」の最後の詰め。

### いま閉じるべき論点
1. **計測の穴**: ファネルの中間イベントが欠損しており、β検証で「どこで離脱しているか」が見えない
2. **安全性の隙間**: content-type未検証、高リスク業種のホールドバックなし
3. **企業リサーチの浅さ**: 業界分類・規制フラグがStep 1で確定せず、Step 3-4の品質に影響

---

## 優先度定義

| 優先度 | 定義 | 判断基準 |
|--------|------|----------|
| P0 | 今スプリント必須 | 「共有される価値」「安全に動く」「β検証に必要」のいずれかに直結 |
| P1 | 次スプリント推奨 | Phase 0.5の品質を高めるが、β開始には不要 |
| P2 | Phase 1以降 | 将来の拡張・改善 |

## 実装順序

### Step 1: イベントタイプ拡充 (P0) — 最優先

**なぜ最初か**: β検証のセンターピンは「共有URL生成率」。計測なくして検証なし。

**やること**:
- EventType に `url_submitted`, `brief_opened`, `share_url_opened`, `return_visit`, `analysis_timeout`, `analysis_empty_dom` を追加
- API route と UI から適切にイベントを発火

**依存**: なし
**完了条件**: 全イベントが logEvent で記録可能

### Step 2: content-type 検証 (P0)

**なぜ次か**: セキュリティの隙間はβ前に閉じるべき。

**やること**:
- fetchWithSSRFProtection の返却後に content-type を検証
- HTML/text 以外（PDF、画像、バイナリ等）のURLは分析を拒否し、エラーメッセージを返却
- 対応テスト追加

**依存**: なし
**完了条件**: 非HTML URLの分析が明確なエラーで拒否される

### Step 3: 企業リサーチ構造化強化 (P0)

**なぜ次か**: 分析品質の基盤。Step 1(企業リサーチ)の出力が貧弱だと、Step 3-4(診断+依頼パック)の業界コンテキストが不足し、汎用的な分析に堕する。

**やること**:
- CompanyResearchResult に `industry_category`, `regulatory_flags`, `business_model_hint` を追加
- HTML meta + URLパターン + テキスト解析で業界分類を推定
- 薬機法/景表法対象業種フラグを自動検知
- 規制フラグを Claude API プロンプトに連携

**依存**: types.ts の型拡張
**完了条件**: 健康食品/化粧品/医薬品URLで regulatory_flags が true

### Step 4: カテゴリ別ホールドバック (P0)

**なぜ次か**: 法令リスクの制御。高リスク業種を検知しても、現状はそのまま分析が進む。HOLDメカニズムが必要。

**やること**:
- `company-research.ts` の規制フラグを基に、高リスク判定時に `regulatory_hold: true` を AnalyzeResponse に追加
- 分析は実行するが、結果に「法令リスクに関する注意」を明示
- UI側で警告バナーを表示

**依存**: Step 3 (企業リサーチ強化)
**完了条件**: 健康食品LPで regulatory_hold フラグが立ち、UI に警告表示

### Step 5: テスト追加 (P0)

**やること**:
- 新規イベントタイプのテスト
- content-type 検証のテスト
- 企業リサーチ構造化のテスト
- ホールドバックのテスト

**依存**: Step 1-4
**完了条件**: 全テスト合格

### Step 6: 計画文書完成 (P0)

**やること**:
- implementation_gap_audit.md ← 完了
- critical_path_plan.md ← 完了
- edge_case_catalog.md
- quality_rubric.md

**依存**: なし
**完了条件**: 4文書が docs/ に存在

| リスク | 確率 | 影響 | 軽減策 |
