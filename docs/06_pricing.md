## 1. 価格設計方針

### 1.1 v2.0→v3.0の転換

| | v2.0 | v3.0 |
|--|------|------|
| 課金モデル | Layer 0-3の段階的開放（連携深度ベース） | **Free/Starter/Pro/Businessの深さ課金** |
| 課金の主動機 | SoR蓄積＋データ連携 | **結果の「深さ」**（概要→詳細→市場→事業） |
| 無料枠 | Layer 0：5回/日、アカウント不要 | **Free：5回/月、アカウント不要** |
| UI思想 | Chrome拡張のLayer昇格 | **Webダッシュボードのタブ開放** |
| 参考モデル | Figma的なフリーミアム | **Ahrefs的な深さ課金** |

### 1.2 課金の力学

「続きを見たい」の連鎖が課金を駆動する。

```
Free：課題がわかる → 「じゃあどう直すの？」は見えない → Starter
Starter：直し方と広告がわかる → 「この市場は伸びてるの？」は見えない → Pro
Pro：市場と流入がわかる → 「チームで共有したい」 → Business
```

## 2. プラン構成

### Free — ¥0

| 項目 | 内容 |
|------|------|
| タブ1（LP分析） | 課題一覧（**概要のみ**）月5回 |
| タブ2-6 | ロック |
| 共有URL | 生成可能（Read-Only概要） |
| アカウント | 不要 |
| 依頼書 | — |
| CSV | — |

**目的**：URLを入れるだけで「おっ」体験。共有URLを通じたバイラルの起点。

### Starter — ¥4,980/月

| 項目 | 内容 |
|------|------|
| タブ1（LP分析） | **詳細**＋依頼書＋薬機法チェック、月30回 |
| タブ2（広告訴求） | Google/Meta訴求文 |
| タブ3-6 | ロック |
| 共有URL | 生成可能（詳細含む） |
| アカウント | 必須（Googleアカウント認証） |
| 依頼書 | デザイナー/エンジニア向け自動生成 |
| CSV | 広告訴求文エクスポート（Google Ads Editor互換） |

**目的**：「どう直すか」と「広告をどう出すか」までが見える。依頼書が外に出る→バイラル。

### Pro — ¥14,800/月

| 項目 | 内容 |
|------|------|
| タブ1（LP分析） | **月200回**＋計測設計 |
| タブ2（広告訴求） | ＋ターゲット推薦＋全広告媒体 |
| タブ3（市場分析） | GSC連携＋検索ボリューム＋市場傾向 |
| タブ4（流入分析） | GA4連携＋参照元＋媒体別 |
| タブ5（競合分析） | 3社まで |
| タブ6 | ロック |
| 自動レポート | 週次メールレポート |

**目的**：「市場はどうなっているか」「流入はどこから来ているか」「競合はどう戦っているか」まで見える。レポートが組織内を回る→横展開。

### Business — ¥49,800/月

| 項目 | 内容 |
|------|------|
| タブ1-5 | 全機能 |
| タブ6（事業分析） | 市場規模推定＋事業モデル＋投資対効果 |
| チーム | 複数メンバー（追加 ¥8,000/名） |
| 複数サイト | 管理サイト数無制限 |
| 競合分析 | 無制限 |
| API/MCP | 外部連携API＋MCP対応 |

**目的**：「チームのデフォルトツール」になる。七條型の「インターン3人分」。

## 4. 課金アップグレードのトリガー設計

### 4.1 Free→Starter

| トリガー | 表示場所 | メッセージ |
|---------|---------|-----------|
| タブ1で「詳細」が見たくなった | 課題カードの「詳細を見る」 | 「Starterプランで詳細＋依頼書が見られます」 |
| タブ2（広告訴求）をクリック | ロックタブ | 「LP分析の訴求をそのまま広告に。Starterで解放」 |
| 月5回を使い切った | URL入力時 | 「今月の無料分析は終了。Starterで月30回」 |
| 薬機法チェック結果が見たい | ロックセクション | 「健康食品/化粧品LPの薬機法チェック。Starterで」 |

### 4.2 Starter→Pro

| トリガー | 表示場所 | メッセージ |
|---------|---------|-----------|
| タブ3-4をクリック | ロックタブ | 「GSC/GA4データで市場と流入が見える。Proで解放」 |
| 月30回を使い切った | URL入力時 | 「Proプランで無制限分析」 |
| 競合分析タブをクリック | ロックタブ | 「競合のLP構造を比較。Proで解放」 |
| 計測設計が見たい | ロックセクション | 「改善の効果を計測する設計。Proで」 |

### 4.3 Pro→Business

| トリガー | 表示場所 | メッセージ |
|---------|---------|-----------|
| タブ6をクリック | ロックタブ | 「事業分析で全体像。Businessで解放」 |
| チーム招待を試みる | 設定画面 | 「チームメンバーを追加。Businessプランで」 |
| API/MCPを試みる | 設定画面 | 「外部連携API。Businessプランで」 |

### 4.4 やらないこと

- 機能を使おうとした瞬間にモーダルで遮断（代わりにロックタブの説明画面を表示）
- 分析結果を一部隠して「Proで全部見る」にする（Free=概要、Starter=詳細。隠すのではなく深さの違い）
- 毎回の分析で課金を促す（トリガーは上記の自然なタイミングのみ）

## 5. 粗利試算

### 5.1 APIコスト実績値（Claude Sonnet 4.6）

| 分析種別 | 入力トークン | 出力トークン | コスト/回 |
|---------|------------|------------|----------|
| タブ1 LP分析（Step 3-4） | ~4,000 | ~3,500 | ≈¥9 |
| タブ2 広告訴求 | ~3,500 | ~3,000 | ≈¥8 |
| フル分析（タブ1+2） | ~7,500 | ~6,500 | ≈¥17 |

※Step 1（企業リサーチ）はHTML fetch+正規表現のみ。Claude API呼び出しなし
※Sonnet 4.6: $3/MTok入力, $15/MTok出力（1$=¥150換算）

### 5.2 プラン別粗利（実績コストベース）

| プラン | 月額 | Claude API原価/月 | Stripe手数料(3.6%) | インフラ按分 | 粗利 | 粗利率 |
|--------|------|------------------|-------------------|------------|------|--------|
| Starter | ¥4,980 | ¥510（30回×¥17） | ¥179 | ¥200 | ¥4,091 | 82% |
| Pro | ¥14,800 | ¥3,400（200回×¥17） | ¥533 | ¥500 | ¥10,367 | 70% |
| Business | ¥49,800 | ¥8,500（500回×¥17） | ¥1,793 | ¥1,000 | ¥38,507 | 77% |

※インフラ: Vercel Pro $20/mo + Supabase Pro $25/mo ≈ ¥6,750/mo をユーザー数で按分

### 5.3 コスト上限に基づく使用制限

各プランが黒字を維持できる月間分析回数の上限：

| プラン | 月額 | 変動費/回 | 固定費 | 損益分岐回数 | 設定上限 | 粗利率@上限 |
|--------|------|----------|--------|------------|---------|------------|
| Free | ¥0 | ¥17 | — | 0（赤字許容） | 5回/月 | -（CAC） |
| Starter | ¥4,980 | ¥17 | ¥379 | 271回 | **30回/月** | 82% |
| Pro | ¥14,800 | ¥17 | ¥1,033 | 810回 | **200回/月** | 70% |
| Business | ¥49,800 | ¥17 | ¥2,793 | 2,766回 | **500回/月** | 77% |

※Freeの赤字（最大¥85/ユーザー/月）はCAC（顧客獲得コスト）として許容
※損益分岐回数は理論値。設定上限は十分なマージンを確保した実用値

### 5.4 APIコスト最適化（Phase 2以降）

- **プロンプトキャッシュ**: Anthropicキャッシュ機能で最大90%削減（同一セッション内）
- **Batch API**: 非同期処理で50%コスト削減（レポート生成等）
- **モデルルーティング**: 企業リサーチはHaiku、診断はSonnet（▲40%コスト）
