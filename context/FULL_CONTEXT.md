### 七條型（マーケ責任者）

## Progressive Value Architecture

```
Layer 0: インストール→1クリック→30秒で3提案（アカウント不要、連携不要）
Layer 1: アカウント作成→ログ蓄積→精度向上
Layer 2: GSC連携→クエリベース分析
Layer 3: GA4連携→自動トラッキング→レポート
```

---

## 構造的矛盾7つ（ゼロベース再設計の根拠）

1. **AIがページ本文を読んでいない** — content-scriptがmeta/h1-h3/CTA/フォームのみ抽出
2. **「公開判断」フレーミングのミスマッチ** — ユーザーは「門番」でなく「推進力」を求めている
3. **PASS/FAILが精度70%で成立しない** — 合否判定は高精度必須
4. **即時反映が核ではない** — 最もペインが深いのは「人に渡す」工程（ハンドオフパック）
5. **7画面フローが360px Side Panelに収まらない** — 画面数の圧縮が必要
6. **ログ構造がMOATにならない** — 判定+理由ではなく、Before/After+結果のラボノート形式が必要
7. **課金ゲートの位置が最適でない** — ログ永続化ではなく「依頼パックのコピー」に置くべき

## API設計

### Cloudflare Workers エンドポイント

| Method | Path | 説明 | 認証 |
|--------|------|------|------|
| POST | `/api/v1/analyze` | ページ分析+提案生成 | 任意 |
| POST | `/api/v1/handoff` | 依頼パック生成 | 任意 |
| POST | `/api/v1/memo` | 承認メモ生成 | 任意 |
| GET | `/api/v1/gsc/queries` | GSCクエリ取得 | OAuth必須 |
| GET | `/api/v1/ga4/metrics` | GA4指標取得 | OAuth必須 |

|------|------|------|
