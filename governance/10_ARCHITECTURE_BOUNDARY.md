## Phase 0.5 必須体験の定義

> 2026-03-15 確定。判断記録: DEC-002, DEC-003。

### コア体験（必須）— これがないとβ検証不能

センターピン「藤野型が分析結果URLをクライアントに送ったか」に直結する5ステップ。

| # | 体験ステップ | 実装 | バックログ | ステータス |
|---|-------------|------|-----------|-----------|
| 1 | URL入力で分析開始 | `UrlInput.tsx` → `/api/analyze` → `analyzer.ts` | B-001 | DONE |
| 2 | 課題構造の提示（企業理解 + ページ読取 + 改善ポテンシャル） | `AnalysisResult.tsx`（上部3カード + 改善ポテンシャル） | B-002 | DONE |
| 3 | 重要Issueの提示（priority上位のissue一覧） | `IssueCard.tsx`（アコーディオン展開、impact: high/medium/low） | B-002 | DONE |
| 4 | 依頼文の提示 | `BriefPanel.tsx`（目的・方向性・具体変更・制約・QA） | B-002 | DONE |
| 5 | 共有 | `ShareButton.tsx` → `/api/share` → `/share/[id]`（PG導線CTA付き） | B-003 | DONE |

**用語の対応:**
- 「課題構造」= `company_understanding` + `page_reading` + `improvement_potential`
- 「重要Issue」= `issues` 配列のpriority上位。新しい情報構造UIは不要

### 強化体験（あれば強い）— βの質を上げるがなくても検証可能

| 体験ステップ | 現状 | 対応案 | 承認要否 |
|-------------|------|--------|---------|
| 施策案の独立表示 | `issue.brief.direction` に内包 | issueカード内に「施策」セクション追加 | UIコピー承認要 |
| バックログ表示 | 未実装（issueがフラットリスト） | issueを「やること一覧」として再構成するビュー | 新規コンポーネント承認要 |
| 再訪導線強化（Web側） | Chrome拡張あり。Web側はURLブックマークのみ | 分析完了後に「ブックマーク推奨」or「Chrome拡張案内」バナー | UIコピー承認要 |
| 有料差分価値の認知拡充 | タブ2-6ロック表示（ツールチップあり） | ツールチップを「得られる価値」中心に拡充 | UIコピー承認要 |

### 今は不要 — Phase 0.5 では作らない

アカウント登録/ログイン、メール通知、CSVエクスポート、決済連携、タブ2-6実装、分析履歴の永続化（Supabase）、チーム機能。

## LP / Web / Chrome拡張の責務分離

> 2026-03-15 確定。判断記録: DEC-002。

### 責務マトリクス

| 責務 | Webダッシュボード (Next.js) | Chrome拡張 (Side Panel) | 備考 |
|------|---------------------------|------------------------|------|
| 初回体験の全て | **主** | — | Webで完結 |
| URL入力 | **主** | **副**（現在のタブURL自動入力） | |
| 分析結果の全表示 | **主** | — | 拡張は結果表示しない or Webに遷移 |
| 共有URL生成 | **主** | — | |
| 共有URL閲覧 | **主** | — | |
| 有料差分の認知 | **主** | — | |
| OGP / バイラル | **主** | — | |
| 再訪の起点 | — | **主** | ブラウザに常駐 |
| 「今見てるLP」の即時分析 | — | **主** | Side Panelのショートカット |

### 責務の原則

| 原則 | 説明 |
|------|------|
| Webが正 | 全ての体験はWebダッシュボードで完結する。Chrome拡張はショートカット |
| 拡張は薄く | Chrome拡張は「分析開始」のトリガーのみ。結果表示はWebに遷移してもよい |
| APIは共通 | 分析パイプラインのコアロジックは一箇所（`src/dashboard/lib/`）。Proxy（`src/proxy/`）はCORS解決のみ |
| 状態はWeb側 | in-memoryストア（将来Supabase）はNext.js側。Chrome拡張は状態を持たない |

### やらないこと

| 層 | やらないこと |
|---|-------------|
| Webダッシュボード | ブラウジング中の割り込み、Chrome拡張のインストール強制 |
| Chrome拡張 | 結果の全表示、共有URL生成、有料訴求、OGP |
