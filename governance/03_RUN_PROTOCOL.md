# 03_RUN_PROTOCOL.md — 毎Run実行プロトコル

> Claude Code が毎回の実装Runで従うべき手順。
> 着手前・実装中・完了時の3段階で構成。

---

## 1. 着手前（必須）

### 読むファイル（この順番で）

1. `CLAUDE.md` — プロダクト定義・セキュリティ要件
2. `governance/00_PROJECT_CHARTER.md` — 最上位方針・Phase 0.5成功条件
3. `governance/05_DO_NOT_DECIDE.md` — 独断禁止リスト
4. `governance/01_PHASE_0_5_BACKLOG.md` — 現在のバックログ状態
5. タスク指示（人間から与えられた今回の指示）

### 着手前テンプレートの提出

コード変更前に、以下を必ず出力する（`TASK_PROMPT_TEMPLATE.md` 参照）:

```
### Scope
- 今回やること
- 今回やらないこと
- Phase 0.5 のどの成功条件にどう効くか

### Assumptions
- 置いている前提

### Decision Required
- 人間判断が必要な論点（あれば）

### Validation Plan
- 完了をどう検証するか

### Open Questions
- 不明点
```

## 2. 実装中

### 守ること

- `05_DO_NOT_DECIDE.md` に該当する判断は独断しない。`02_DECISION_LOG.md` に記録し、人間にエスカレーションする
- 1 Run = 1関心事。スコープを自己膨張させない
- 「ついでにやっておく」は禁止。指示されたことだけやる
- 新規ファイル作成は最小限。既存ファイルの編集を優先する
- 新規依存・新規テーブル・新規APIの追加は人間承認が必要
- UIコピー（ボタンラベル、見出し、エラーメッセージ等）は `08_UI_COPY_LOCKED.md` を参照し、変更しない
- 仕様書（`docs/` 配下）を勝手に更新しない

### 中間チェック

大きな変更（ファイル3つ以上 or 100行以上）の場合、中間で以下を出力する:

```
### 中間レポート
- ここまでの変更内容
- 残りの作業
- 懸念点（あれば）
```

## 3. 完了時（必須）

### 完了報告の提出

`COMPLETION_REPORT_TEMPLATE.md` に従い、以下を出力する:

```
### Completion Report
- What changed: 変更したファイルと内容
- Why this matters: Phase 0.5 成功条件との関係
- Validation: テスト結果・動作確認結果
- What I did not change: 意図的に変更しなかったもの
- Risks: 残存リスク
- Suggested next step: 次にやるべきこと（提案のみ。勝手にやらない）
```

### バックログ更新

- 完了したタスクのステータスを `DONE` に更新する
- 新たに発見したタスクは `02_DECISION_LOG.md` に記録し、バックログへの追加は人間に提案する

---

## 禁止事項（再掲）

1. 着手前テンプレートなしでコード変更を開始しない
2. `05_DO_NOT_DECIDE.md` の判断領域を独断しない
3. スコープを自己膨張させない
4. 仕様書を勝手に更新しない
5. 完了報告なしでRunを終了しない
6. テスト通過のみで完了とせず、実動作への影響を報告する
