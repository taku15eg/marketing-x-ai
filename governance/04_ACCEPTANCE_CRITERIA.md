## 全実装共通の受け入れ条件

### 必須（これを満たさないと完了ではない）

- [ ] TypeScript型エラーがゼロ
- [ ] 既存テストが全てPASS
- [ ] セキュリティテスト（SEC-1〜SEC-13）が全PASS
- [ ] `05_DO_NOT_DECIDE.md` に該当する独断判断をしていない
- [ ] 新規依存を追加していない（追加した場合は人間承認済み）
- [ ] UIコピーを変更していない（変更した場合は人間承認済み）
- [ ] `CLAUDE.md` のセキュリティ要件に違反していない
- [ ] 完了報告テンプレートを提出済み

### 品質（可能な限り満たす）

- [ ] 変更した機能の実動作確認手順を明記
- [ ] エッジケースへの対応を記述
- [ ] パフォーマンスへの影響がないことを確認

## 機能別の受け入れ条件

### URL入力→分析結果表示

- [ ] URLを入力して30秒以内に結果が表示される
- [ ] 4ステップ進捗が表示される
- [ ] 企業理解→ページ読取→診断→依頼パックの順に処理される
- [ ] 結果JSONが `CLAUDE.md` の出力JSON構造に準拠

### 共有URL

- [ ] 共有URLで第三者が結果を閲覧できる
- [ ] IDがUUID/nanoid 21文字以上（連番・タイムスタンプ禁止）
- [ ] OGPに分析対象URLそのものが含まれない
- [ ] 「Powered by Publish Gate」が表示される

### Chrome拡張

- [ ] permissions が `activeTab`, `sidePanel` のみ
- [ ] LP閲覧時にSide Panelで分析開始できる
- [ ] PIIマスキングが動作する
- [ ] content-scriptがオンデマンド注入

### セキュリティ

- [ ] SSRF防御: プライベートIPレンジが全て拒否される
- [ ] DNS resolve後のIPもチェックされる
- [ ] リダイレクト先もチェックされる（最大3回）
- [ ] プロンプトインジェクション防御が実装されている
- [ ] APIキーがクライアントに露出しない
- [ ] `dangerouslySetInnerHTML` 未使用

```
1. Claude Code が完了報告を提出
2. 上記チェックリストに照らして自己チェック
3. 人間がレビュー
4. 指摘があれば修正 → 再提出
5. 人間が承認 → DONE
```
