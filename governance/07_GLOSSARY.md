# 07_GLOSSARY.md — 用語集

> プロジェクト内で使う用語の定義。認識のズレを防ぐ。

---

## プロダクト用語

| 用語 | 定義 |
|------|------|
| Publish Gate (PG) | 本プロダクトの名称。URLを入れるだけでLP改善レポートが得られるSaaS |
| ダッシュボード | Webブラウザで使う本体UI。Ahrefs Site Explorerと同じ体験パターン |
| Side Panel | Chrome拡張のUI。ブラウザ右側に常駐するパネル |
| 分析結果 | 4ステップパイプラインの出力。課題構造・依頼パックを含むJSON |
| 共有URL | 分析結果を第三者と共有するための公開URL |
| 依頼パック / 依頼書 | デザイナー・エンジニア向けの構造化ブリーフ。コピー文言は含まない |
| タブ1〜6 | ダッシュボードの6つの分析カテゴリ。Phase 0.5ではタブ1のみ |

## 分析エンジン用語

| 用語 | 定義 |
|------|------|
| 4ステップパイプライン | ①企業理解→②ページ読取→③診断→④依頼パック生成 |
| FV (First View) | ページを開いた時にスクロールなしで見える領域 |
| CTA | Call To Action。ユーザーに行動を促すボタン・リンク |
| Vision API | Claude の画像認識機能。FVスクリーンショットの分析に使用 |

## ユーザーアーキタイプ

| 用語 | 定義 |
|------|------|
| 藤野型 | 代理業。案件スケーリングが最大価値。Phase 0.5の第1ターゲット |
| 浦田型 | 事業会社マーケ。根因特定+自動ログが最大価値 |
| 七條型 | マーケ責任者。チーム標準化が最大価値 |

## Phase用語

| 用語 | 定義 |
|------|------|
| Phase 0.5 | β検証フェーズ。タブ1+共有URL+Chrome拡張。5名で検証 |
| センターピン | 各Phaseの移行を判定する1つの問い |
| β | Phase 0.5の検証対象ユーザー群（藤野型5名） |

## 運用用語

| 用語 | 定義 |
|------|------|
| Run | Claude Code の1回の実行単位。1 Run = 1関心事 |
| 着手前テンプレート | Run開始時に提出するScope/Assumptions/Decision Required等 |
| 完了報告 | Run終了時に提出するWhat changed/Validation等 |
| エスカレーション | 独断禁止リストに該当する判断を人間に委ねること |
| 正本 | 各種情報の信頼できる唯一のソース |

## 技術用語

| 用語 | 定義 |
|------|------|
| SSRF | Server-Side Request Forgery。サーバー側のfetch悪用攻撃 |
| RLS | Row Level Security。Supabaseの行レベルアクセス制御 |
| Manifest V3 | Chrome拡張の最新仕様 |
| Side Panel API | Chrome拡張でブラウザ横にパネルを表示するAPI |
