# 06_REVIEW_CHECKLIST.md — レビューチェックリスト

> 各実装Runの完了時に、Claude Code が自己チェックし、人間がレビューするためのリスト。

---

## スコープチェック

- [ ] 指示された範囲だけを変更したか
- [ ] 「ついでに」の変更をしていないか
- [ ] Phase 0.5 成功条件に直結する変更か
- [ ] 新機能の先回り実装をしていないか

## 独断判断チェック

- [ ] `05_DO_NOT_DECIDE.md` の禁止領域に触れていないか
- [ ] UIコピーを変更・追加していないか
- [ ] TTL・制限値・課金条件を変更していないか
- [ ] 新規依存を追加していないか
- [ ] 新規テーブル・APIを追加していないか
- [ ] 仕様書（`docs/`配下）を更新していないか

## セキュリティチェック

- [ ] SSRF防御が維持されているか
- [ ] APIキーがクライアントに露出していないか
- [ ] `dangerouslySetInnerHTML` を使っていないか
- [ ] プロンプトインジェクション防御が維持されているか
- [ ] 共有URL IDが推測不可能か
- [ ] `.env.local` が `.gitignore` に含まれているか

## コード品質チェック

- [ ] TypeScript型エラーがゼロか
- [ ] 既存テストが全てPASSか
- [ ] 変更に対応するテストを追加したか
- [ ] エラーハンドリングが適切か

## 整合性チェック

- [ ] `CLAUDE.md` の仕様と矛盾していないか
- [ ] 出力JSON構造が仕様に準拠しているか
- [ ] 既存のコンポーネント・関数との整合性があるか

## 実動作チェック

- [ ] テスト通過だけでなく、実動作への影響を確認したか
- [ ] エッジケース（無効URL、タイムアウト、404等）を考慮したか
- [ ] パフォーマンスへの影響を確認したか

## ドキュメントチェック

- [ ] 完了報告テンプレートを提出したか
- [ ] バックログのステータスを更新したか
- [ ] 判断が必要だった場合、`02_DECISION_LOG.md` に記録したか

---

```
### Run: [日付] [タスク概要]

- 指摘事項: [あれば]
- 修正要否: [あれば]
```
