# 05_DO_NOT_DECIDE.md — AIが独断で決めてはいけないもの

> このリストに該当する判断を求められた場合、Claude Code は以下を行う:
> 1. 判断を保留する
> 2. `02_DECISION_LOG.md` に「判断保留・人間エスカレーション」として記録する
> 3. 選択肢を最小限（2-3個）で提示する
> 4. 人間の回答を待つ

---

## 独断禁止リスト

### UIコピー・文言

- ボタンラベル、見出し、説明文、プレースホルダー
- エラーメッセージの文言
- OGP title / description
- ローディング中のメッセージ
- 通知・メールの文面
- ロック表示の文言（「Proプランで解放」等）

> 現在確定済みのUIコピーは `08_UI_COPY_LOCKED.md` を参照。

### 制限値・パラメータ

- TTL（キャッシュ有効期限）
- レート制限の具体値（月N回、分N回）
- レスポンスサイズ上限
- タイムアウト値
- リトライ回数・間隔
- 課金条件・価格

> 現在の確定値は `CLAUDE.md` に記載。変更は人間が判断する。

### 法務・コンプライアンス

- 薬機法チェックの判定基準・閾値
- 景品表示法の判定基準
- プライバシーポリシーの文言
- 利用規約の文言
- 個人情報の取り扱い方針

### API設計方針

- エンドポイントの新設
- レスポンス構造の変更
- 認証方式の変更
- APIバージョニング方針

### データベース

- テーブルの新設
- カラムの追加・削除
- RLSポリシーの変更
- マイグレーション

### 依存関係

- npm パッケージの新規追加
- 既存パッケージのメジャーバージョンアップ
- 外部サービスとの新規連携

### 分析精度

- プロンプトの構造的変更（微調整は許可）
- 分析ステップの追加・削除
- 出力JSON構造の変更
- スコアリング基準の変更

### 仕様書

- `docs/` 配下のファイル内容の更新
- `CLAUDE.md` の内容の更新
- `INSTRUCTION.md` の内容の更新

---

## Claude Code に許可されている判断

- バグ修正のための実装方法の選択
- テストの追加・修正
- コード内のコメント追加
- リファクタリング（外部仕様を変えない範囲）
- 型定義の修正
- lint / format の修正
- ガバナンスファイルのステータス更新（バックログのDONE化等）
- `02_DECISION_LOG.md` への記録追加
