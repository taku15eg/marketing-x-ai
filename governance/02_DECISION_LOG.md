# 02_DECISION_LOG.md — 判断ログ

> 実装中に発生した判断を記録する。
> 人間が判断したもの、Claude Code が判断を仰いだもの、両方を残す。
> 「なぜそうなったか」を後から追跡できることが目的。

---

## 記録フォーマット

```
### DEC-XXX: [判断タイトル]
- **日付**: YYYY-MM-DD
- **起票者**: 人間 / Claude Code
- **判断者**: 人間 / Claude Code（許可範囲内）
- **論点**: 何を決める必要があったか
- **選択肢**: A / B / C
- **決定**: 選んだもの
- **理由**: なぜそれを選んだか
- **影響範囲**: 何に影響するか
- **関連バックログ**: B-XXX
```

---

## 判断記録

### DEC-001: ガバナンスファイルの配置場所
- **日付**: 2026-03-14
- **起票者**: Claude Code
- **判断者**: 人間（タスク指示内で承認）
- **論点**: ガバナンスファイルを `.governance/`（dotディレクトリ）vs `governance/`（通常ディレクトリ）のどちらに置くか
- **選択肢**: A) `.governance/` / B) `governance/`
- **決定**: B) `governance/`
- **理由**: dotディレクトリはツールが無視しがち。視認性と参照しやすさを優先
- **影響範囲**: 全ガバナンスファイルのパス
- **関連バックログ**: なし（ガバナンス基盤整備）

---

> 新しい判断は上記フォーマットに従い、DEC-XXX の連番で追記する。
> Claude Code が `05_DO_NOT_DECIDE.md` に該当する判断を求められた場合、このログに「判断保留・人間エスカレーション」として記録する。
