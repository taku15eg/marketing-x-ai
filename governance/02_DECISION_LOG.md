# 02_DECISION_LOG.md — 判断ログ

> 実装中に発生した判断を記録する。
> 人間が判断したもの、Claude Code が判断を仰いだもの、両方を残す。
> 「なぜそうなったか」を後から追跡できることが目的。

---

## 記録フォーマット

```
### DEC-XXX: [判断タイトル]
- **日付**: YYYY-MM-DD
- **起票者**: 人間 / Claude Code
- **判断者**: 人間 / Claude Code（許可範囲内）
- **論点**: 何を決める必要があったか
- **選択肢**: A / B / C
- **決定**: 選んだもの
- **理由**: なぜそれを選んだか
- **影響範囲**: 何に影響するか
- **関連バックログ**: B-XXX
```

---

## 判断記録

### DEC-001: ガバナンスファイルの配置場所
- **日付**: 2026-03-14
- **起票者**: Claude Code
- **判断者**: 人間（タスク指示内で承認）
- **論点**: ガバナンスファイルを `.governance/`（dotディレクトリ）vs `governance/`（通常ディレクトリ）のどちらに置くか
- **選択肢**: A) `.governance/` / B) `governance/`
- **決定**: B) `governance/`
- **理由**: dotディレクトリはツールが無視しがち。視認性と参照しやすさを優先
- **影響範囲**: 全ガバナンスファイルのパス
- **関連バックログ**: なし（ガバナンス基盤整備）

### DEC-002: INSTRUCTION.md と governance/ の責務分離
- **日付**: 2026-03-14
- **起票者**: Claude Code
- **判断者**: 人間
- **論点**: `INSTRUCTION.md` と `governance/` の責務が一部重複する。統合するか分離するか
- **選択肢**: A) 併存 / B) governance/ に統合 / C) 役割を明文化して分離
- **決定**: C) 役割を明文化して分離
- **理由**: INSTRUCTION.md = 実行導線（ビルド手順）、governance/ = 判断規律・正本。それぞれ異なる目的を持つ
- **影響範囲**: INSTRUCTION.md 冒頭に正本宣言を追加
- **関連バックログ**: なし（ガバナンス基盤整備）

### DEC-003: UIコピー正式承認レビュー
- **日付**: 2026-03-14
- **起票者**: Claude Code
- **判断者**: 保留（人間レビュー待ち） — **OPEN**
- **論点**: `08_UI_COPY_LOCKED.md` に記載された150+のUIコピーを正式承認するかどうか
- **選択肢**:
  - A) 現状の全コピーを一括で正式ロックする
  - B) カテゴリ別にレビューし、段階的に正式ロックする
  - C) β検証後のフィードバックを踏まえてからレビューする
- **決定**: 未決定（OPEN）
- **理由**: 現在は暫定ロック状態。Claude Code による独断変更は禁止だが、人間による正式承認は未了
- **影響範囲**: 全UIコピー。正式承認後は変更手続きがより厳格になる
- **関連バックログ**: なし（ガバナンス運用）
