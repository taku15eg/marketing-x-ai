# 02_DECISION_LOG.md — 判断ログ

> 実装中に発生した判断を記録する。
> 人間が判断したもの、Claude Code が判断を仰いだもの、両方を残す。
> 「なぜそうなったか」を後から追跡できることが目的。

---

## 記録フォーマット

```
### DEC-XXX: [判断タイトル]
- **日付**: YYYY-MM-DD
- **起票者**: 人間 / Claude Code
- **判断者**: 人間 / Claude Code（許可範囲内）
- **論点**: 何を決める必要があったか
- **選択肢**: A / B / C
- **決定**: 選んだもの
- **理由**: なぜそれを選んだか
- **影響範囲**: 何に影響するか
- **関連バックログ**: B-XXX
```

## 判断記録

### DEC-001: ガバナンスファイルの配置場所
- **日付**: 2026-03-14
- **起票者**: Claude Code
- **判断者**: 人間（タスク指示内で承認）
- **論点**: ガバナンスファイルを `.governance/`（dotディレクトリ）vs `governance/`（通常ディレクトリ）のどちらに置くか
- **選択肢**: A) `.governance/` / B) `governance/`
- **決定**: B) `governance/`
- **理由**: dotディレクトリはツールが無視しがち。視認性と参照しやすさを優先
- **影響範囲**: 全ガバナンスファイルのパス
- **関連バックログ**: なし（ガバナンス基盤整備）

### DEC-002: 戦略再設計 — センターピン指標の変更
- **日付**: 2026-03-15
- **起票者**: Claude Code（戦略再設計タスク内）
- **判断者**: 人間エスカレーション中（提案段階）
- **論点**: Phase 0.5のセンターピン指標を「共有URL生成」から「診断品質＋行動変容」に変更するか
- **選択肢**: A) 診断品質＋行動変容に変更 / B) 共有URL生成を維持 / C) 両方を同列
- **決定**: A) を提案。人間の最終承認待ち
- **理由**: インタビューが示す最大のペインは「なぜその数字かわからない」「何をすればいいかわからない」であり、共有はペイン解消の結果。品質が伴わなければ共有は起きない。センターピンは原因（品質）であるべきで結果（共有）ではない
- **影響範囲**: governance/00_PROJECT_CHARTER.md, governance/09_METRICS_SPEC.md, docs/09_beta_plan.md, docs/08_roadmap.md
- **関連バックログ**: 全P0タスク

### DEC-003: 戦略再設計 — Chrome拡張のPhase 1移動
- **日付**: 2026-03-15
- **起票者**: Claude Code（戦略再設計タスク内）
- **判断者**: 人間エスカレーション中（提案段階）
- **論点**: Phase 0.5スコープからChrome拡張を除外しPhase 1に移動するか
- **選択肢**: A) Phase 1に移動 / B) Phase 0.5に維持
- **決定**: A) を提案。人間の最終承認待ち
- **理由**: 5名βでChrome拡張の配布・インストール・デバッグにかかる工数 > 得られる学習。Webダッシュボードだけで仮説検証は可能
- **影響範囲**: governance/01_PHASE_0_5_BACKLOG.md (B-004), docs/08_roadmap.md, governance/00_PROJECT_CHARTER.md
- **関連バックログ**: B-004

### DEC-004: 戦略再設計 — 市場参入順序の明示化
- **日付**: 2026-03-15
- **起票者**: Claude Code（戦略再設計タスク内）
- **判断者**: 人間エスカレーション中（提案段階）
- **論点**: 「初手市場」と「長期本命市場」を全文書で明示的に分離するか
- **選択肢**: A) 明示的分離（藤野型=初手、浦田型=本命） / B) 現状の暗黙的分離を維持

- **影響範囲**: CLAUDE.md, governance/00_PROJECT_CHARTER.md, docs/01_requirements.md, docs/07_marketing.md
